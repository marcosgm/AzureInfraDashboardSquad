# Routing Rules

## Domain Routing

| Domain | Primary | Backup |
|--------|---------|--------|
| Architecture, system design, scope | Kranz | — |
| React, UI components, diagrams, charts, dashboard layout | Swigert | — |
| Azure APIs, MCP integration, caching, telemetry, data pipeline | Lovell | — |
| Tests, QA, edge cases, validation | Haise | — |
| Code review, approval gates | Kranz | — |

## File Routing

| Pattern | Agent |
|---------|-------|
| `src/components/**`, `src/pages/**`, `src/hooks/**`, `*.css`, `*.scss` | Swigert |
| `src/api/**`, `src/services/**`, `src/cache/**`, `src/azure/**` | Lovell |
| `**/*.test.*`, `**/*.spec.*`, `tests/**` | Haise |
| `src/config/**`, `docs/**` | Kranz |

## Keywords

| Keyword | Agent |
|---------|-------|
| diagram, chart, graph, UI, component, layout, dashboard, visualization | Swigert |
| Azure, MCP, API, cache, telemetry, metric, polling, refresh | Lovell |
| test, spec, coverage, assertion, edge case, quality | Haise |
| architecture, design, scope, decision, review, structure | Kranz |
